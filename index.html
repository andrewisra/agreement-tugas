🛠️ Anggota 1

Infrastruktur, Auth, & Local DB
  • Setup Proyek: Inisialisasi Kotlin-Android Native (SDK 30-35), routing dasar, kesepakatan 4 halaman XML minimum.
  • Database Lokal: Setup SQLite/Room Database untuk menyimpan status Pinned Family secara lokal.
  • Autentikasi: UI/UX Login, simpan JWT Token, & buat interceptor untuk token expired (409).
  • Profile: Halaman profil (Get/Patch User), fitur Sign Out, dan UI Bottom Sheet Edit Name.
  • Network Sensing: Logika background deteksi internet putus & memunculkan pop-up global.
  • Dokumentasi API: Menyusun file openapi.yml untuk REST API.

👨‍👩‍👧‍👦 Anggota 2

UI/UX Utama & Manajemen Family
  • Navigasi: Implementasi Bottom Navbar (Home, Map, Families) & Header global.
  • Home: Halaman "My Families" dan 5 random "Discover Families" (termasuk fitur truncate text).
  • Families List: Menampilkan daftar seluruh family, filter All/My Families, dan menyambungkan tombol Pin ke Local DB.
  • Family CRUD: Form Create Family (dengan validasi icon), halaman Detail Family, serta menyalin Family Code.
  • Join & Leave: Pop-up input kode untuk Join grup, dan pop-up konfirmasi untuk Leave grup.

🗺️ Anggota 3

Peta, Sensor Perangkat, & WebSocket
  • Integrasi Peta: Pasang library (osmdroid/maplibre), minta izin GPS (permission), tampilkan peta & titik pengguna.
  • Sensor Hardware: Ambil data GPS, persentase Baterai, dan orientasi/kompas HP (derajat arah).
  • WebSocket (Kirim): Connect ke server (dengan token Bearer), jalankan interval 1 detik untuk kirim update_presence.
  • WebSocket (Terima): Tangkap member_presence_updated dan ubah posisi pin anggota lain secara mulus.
  • Logika Peta Dinamis: Hilangkan pin jika tidak ada sinyal > 5 detik (offline), dan buat bottom sheet info detail saat pin diklik.

🏆 Fitur Bonus & Pengerjaan Bersama

Dikerjakan kolaboratif atau dibagikan setelah fitur utama (MVP) berjalan lancar tanpa crash.

  • Search Family (Anggota 2): Tambahkan search bar di atas halaman Families.
  • Internet Status (Anggota 3): Tampilkan status "WiFi" atau "Mobile" pada info peta.
  • Mark Favorite Location (Anggota 1 & 3): Simpan titik khusus di peta ke dalam SQLite lokal.
  • Accessibility Testing (Bersama): Gunakan Accessibility Scanner, perbaiki UI (contrast, touch target), dan screenshot hasilnya.
  • Livestreaming (Opsional Ekstra): Bebas menggunakan library pihak ketiga jika ada waktu luang.
  • Finishing (Bersama): Merapikan README.md, screenshot aplikasi, dan rilis tag di GitHub.
