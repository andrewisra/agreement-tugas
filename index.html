Mekanisme Login, Session, dan Token

1) Setelah login sukses, set session:
$_SESSION["user_id"], $_SESSION["role"], $_SESSION["csrf_token"] = bin2hex(random_bytes(32));

2) Semua form POST wajib menyertakan token:
<input type="hidden" name="csrf_token" value="<?= $_SESSION['csrf_token'] ?>">

3) Validasi token di server untuk tiap POST:
if (!hash_equals($_SESSION['csrf_token'] ?? '', $_POST['csrf_token'] ?? '')) { http_response_code(403); exit('Invalid token'); }

4) Akses halaman by role: guest hanya lihat katalog; buyer yang buka dashboard seller dialihkan ke /home; seller yang buka halaman buyer dialihkan ke /dashboard.

Struktur Folder (plus fungsi tiap folder)

php/src/
 ├─ index.php          entry point + router sederhana
 ├─ controllers/       terima request, panggil model, pilih view
 ├─ models/            query DB, validasi bisnis, helper transaksi
 ├─ views/             tampilan HTML/PHP yang dirender
 ├─ public/
 │   ├─ css/           stylesheet (global.css, page-specific css)
 │   ├─ js/            script (index.js, cart.js, checkout.js)
 │   └─ uploads/       file binary: gambar produk & logo toko
 └─ config/
     └─ db.php         koneksi database (PDO/mysqli)

Penjelasan ringkas

  • index.php — gerbang utama aplikasi. Boleh jadi “router” sederhana pakai ?page=.... Di sini juga start session_start().
  • controllers/ — tempat logika HTTP: baca $_GET/$_POST, cek token, cek role, panggil model, terus render view. Contoh: AuthController.php, ProductController.php.
  • models/ — semua urusan database. Satu file per entitas: User.php, Product.php, Order.php. Di sini juga ada fungsi transaksi: hold balance, kurangi stok.
  • views/ — HTML yang dipakai controller buat output. Boleh bikin views/layouts/navbar.php biar reusable.
  • public/css — kumpulan CSS. Minimal global.css berisi palette dan komponen dasar. Hindari inline style.
  • public/js — file JS per fitur. Contoh: cart.js untuk update quantity via AJAX; checkout.js untuk validasi saldo.
  • public/uploads — penyimpanan gambar lokal. Database hanya simpan path, bukan binary.
  • config/db.php — satu pintu koneksi DB. Pakai env dari docker-compose kalau perlu.
